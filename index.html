1

Sandwich Panel Production Line
Stynergy LLP

2019, Karaganda, Kazakhstan

My first commercial project outside of maintenance service. 137 DI / DO / AO. 12 VFDs (15 in total) controlled by CC-Link IE Field Basic protocol. The project turned out to be difficult. 16 motors, 14 of which must be speed synchronized. The line in the process of sawing off the sandwich panel should not have stopped movement (flying saw). There was no PLC module for the positioning of the flying saw servo drive (it was not included in the project). An encoder was used that measured the length and linear speed of the line. The servo motor speed was controlled by the PLC analog output (according to the measured line speed). It was necessary to obtain the length of the sandwich panel with an error of less than 5 mm from the specified length. With a panel length of 9 m, the error cannot exceed 0.05%. With sweat and blood, we got an error of no more than 3mm. I developed PLC code and HMI project. Did commissioning work.

VFDs parameters indication system
Kazakhmys Corporation LLC

2020, Nurkazgan, Kazakhstan

Commercial project. The goal of the project was to receive data on electrical parameters from 16 VFDs. Mitsubishi VFD only had the RS-485 interface available, so the Modbus RTU protocol was chosen. I used 2 PLCs (8 VFDs each). Outside of the main goal, I also set additional goals for myself to improve my programming skills. The first goal was to implement ALL available network mode functions of the Mitsubishi VFD. For receiving data these are the operating mode, the state of the discrete outputs, analog values (not only electrical parameters, but also, for example, the temperature of the VFD), the model and series of the VFD, the error code (with decoding), VFD parameters values. For sending data, these are writing digital inputs, frequency task, setting VFD parameters, resetting errors. The second goal was to improve my coding skills so that I need less development time in the future. To achieve this, I studied creating and using data structures, using them in arrays, setting up and using auto-filling for memory areas, registering and using system variables to import from PLC to HMI. The third goal was to obtain the most optimized communication algorithm over the Modbus RTU protocol to obtain up-to-date quality data (receive data / writing commands of all 8 inverters in less than 500ms). It was necessary to use each send/request to the maximum and not make useless send/request operations (e.g. send frequency task when the VFD is not running). I developed PLC code and achieved all my and project goals.

Technologies I worked with here: Mitsubishi PLCs FX3U (+GX Works 2, Melsoft Navigator), VFDs FR-A/F800, Modbus RTU.

But SCADA needs a global rework in accordance with the HMI ISA-101 design standard. Thanks to Realpars.com and PAS's "The High Performance HMI" book for covering this topic. I wish I knew about the existence of the standard a few years ago.

Stacker's control system
Kazakhmys Corporation LLC

2019, Nurkazgan, Kazakhstan

Commercial project. The goal of the project was to replace old contactors-relays schemes for two stacker machines on a crusher complex of the Nurkazgan Enrichment Plant. The old scheme was unreliable, had frequent malfunctions and it was hard to diagnose. Two machines are identical and consist of a feeder, conveyor, and stacker. Before this project, we already had massive experience with those stackers because they were under our maintenance service. We replaced old equipment with the VFDs and PLC. PLC controls the process and receives all information (Current / Voltage / Frequency /D iscrete statuses / etc) from VFDs via Mitsubishi Inverter Protocol. All the data were added to the crusher complex operator’s HMI via Profibus DP and data started to be logged. As result, Stacker machines' malfunctions decreased to a minimum and in those infrequent cases it is easy to diagnose a problem with help of all indicated and logged data on the HMI. I developed PLC code and made small changes in the HMI project. Did commissioning work.

Technologies I worked with here: Mitsubishi PLCs FX3U (+GX IEC Developer), VFDs FR-A800, Mitsubishi Inverter Protocol, Profibus DP.

Measurement of length and line speed of the produced cable

2017, Saran, Kazakhstan

I implemented measurement and display of the length of the produced cable and linear speed on two production lines. Before this, line operators relied on the length marks, printed on the cable. The printer works completely autonomously. I used a discrete sensor connected to the high-speed inputs of the PLC. The sensor was detecting the teeth of a part bolted to the gearbox shaft of the wheel capstan machine. Then I made a code with simple math for length and used the SPD operator built into the Mitsubishi PLC for line speed. Made changes to the HMI. For a more accurate measurement, it is better to use an encoder, but since we did not have an extra encoder and the line speed is low enough, I used an inductive sensor.

Technologies I worked with here: Mitsubishi PLCs FX2N (+GX Developer), M2I TOP HMIs (+XDesignerPlus).

Modernization of the "Twisting in a pair" machine

2017, Saran, Kazakhstan

Filled with blind student enthusiasm, I came up with the idea of modernization. The idea was born when I was once again wiping the oil flowed out of a gearbox in our electrical cabinet. The gearbox provided the twisting pitch of the twisted pair of the network cable. I wanted to exclude it from the mechanism and provide the twist pitch using the AI/AO reference between VFDs. I made a reference between 4 VFDs, tuned the PID, tuned the vector control (what a surprise it was for me that the motor cannot move a full drum of cable on 5 Hz without vector control). And... I failed. Without an encoder (which we did not have), it is impossible to get the precise pitch of the twist. It turned out that experienced Chinese designers are smarter than a student who did not have even a year of experience. But I have no regrets and got a great experience.

Technologies I worked with here: YASKAWA VFDs V1000 series (+DriveWizard).
